# Contributing to Clima

Thank you for your interest in contributing to Clima! We welcome all contributions.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/clima.git`
3. Create a feature branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Test your changes in a Home Assistant instance
6. Commit with descriptive messages: `git commit -am 'Add my feature'`
7. Push to your fork: `git push origin feature/my-feature`
8. Submit a Pull Request

## Development Setup

### Requirements

- Python 3.9+
- Home Assistant development environment
- Git

### Testing

Before submitting a PR:

1. Test your changes in a Home Assistant instance
2. Verify all entities are created correctly
3. Test all automations with your setup
4. Check for any error messages in Home Assistant logs

## Code Style

- Follow PEP 8 for Python code
- Use descriptive variable and function names
- Add comments for complex logic
- Keep functions focused and modular

## Commit Messages

- Use clear, descriptive commit messages
- Reference issue numbers when applicable: `Fixes #123`
- Use present tense: "Add feature" not "Added feature"

## Feature Requests

Found a feature you'd like? Open an issue with:
- Clear description of the feature
- Use case and motivation
- Any relevant examples

## Bug Reports

Found a bug? Please report it with:
- Home Assistant version
- Clima version
- Steps to reproduce
- Expected vs. actual behavior
- Error logs (from **Settings → System → Logs**)

## Questions?

Feel free to open an issue for questions about development or submit a discussion post.

Thank you for contributing! 🎉
